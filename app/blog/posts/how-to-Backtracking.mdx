## Backtracking

### When to use ` i = startIndex`, when to use `i = 0`?

For example, given input array [2,3,6].

Case 1: for (int i = startindex)
```
for (int i = startIndex; i < array.length; i++) {
    // ignore
    backtracking(array, sum, i);
    // ignore
}
```
```
                    [ 2 , 3 , 6 ]
               /          |        \
              /           |         \
             /            |          \
            /             |           \
          [2]            [3]           [6] 
       /   |    \       /    \          |
   [2,2] [2,3] [2,6]  [3,3] [3,6]     [6,6]
                            .
                            .
                            .
```

Case2 : for (int i = 0)
```
for (int i = 0; i < array.length; i++) {
    // ignore
    backtracking(array, sum, i);
    // ignore
}
```
```
                       [ 2 , 3 , 6 ]
                  /          |              \
                /            |               \
              /              |                \
            /                |                 \
          [2]               [3]                [6] 
      /    |    \       /    |    \        /    |    \
  [2,2] [2,3] [2,6]  [3,2] [3,3] [3,6]  [6,2] [6,3] [6,6]
                            .
                            .
                            .
```

startIndex来控制for循环的起始位置，对于组合问题，什么时候需要startIndex呢？

如果是一个集合来求组合的话，就需要startIndex，例如：77.组合 (opens new window)，216.组合总和III (opens new window)。

如果是多个集合取组合，各个集合之间相互不影响，那么就不用startIndex，例如：17.电话号码的字母组合
